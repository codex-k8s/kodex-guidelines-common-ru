# Инструкции для агентов

## Назначение репозитория

Этот репозиторий содержит общие инженерные правила, которые можно использовать в разных проектах.

## Правила работы

- Ответы пользователю и оформление PR писать на русском языке.
- Тексты commit message писать на английском языке.
- Прямой push в защищённые ветки запрещён: изменения вносятся через отдельную ветку и PR. Защищёнными считаются `main`, `master` и другие ветки, закрытые правилами branch protection или ruleset.
- Если этот репозиторий используется как внешний пакет в проектах-потребителях, проверить, нужно ли выпустить новую версию пакета и обновить локальные копии или gitlink в этих проектах отдельными PR.
- Репозиторий должен оставаться самодостаточным: не добавлять обязательных ссылок на другие пакеты правил.
- Не добавлять обязательные нормы, завязанные на конкретный проект, продукт, slug репозитория, префикс env, схему входа или частную release/delivery-политику.
- Проектные overlays должны жить в локальной документации проекта-потребителя.
- Примеры допускаются, если они явно помечены как примеры и не превращаются в обязательную норму для всех потребителей.

## Состав репозитория

Документы, общие для backend и frontend (не дублируются в `go/` и `vue/`).

- `check_list.md` — общий чек-лист (дальше — профильные в `go/` и `vue/`).
- `project_architecture.md` — зоны, границы ответственности, структура репо.
- `design_principles.md` — DDD/SOLID/DRY/KISS/Clean Architecture.
- `libraries_reusable_code_requirements.md` — общие правила выноса кода в `libs/*`.
- `external_dependencies_catalog.md` — шаблон каталога внешних библиотек и инструментов.

Конкретные проектные ограничения, технологические стеки и эксплуатационные соглашения должны задаваться локальным overlay проекта-потребителя.
