# Common Engineering Guidelines

Публичный репозиторий общих инженерных правил: архитектурные принципы, чек-листы, правила переиспользуемого кода и шаблон каталога внешних зависимостей.

## Назначение

Этот репозиторий содержит правила, общие для backend и frontend:

- архитектурные принципы;
- общие чек-листы;
- правила переиспользуемого кода;
- шаблон каталога внешних зависимостей.

## Использование

Репозиторий можно использовать:

- как внешний пакет руководящих правил;
- как базовый набор документов для локальной проектной копии, например в `docs/design-guidelines/common/**` или аналогичном каталоге проекта;
- как основу для проектного overlay, где конкретный проект фиксирует свои ограничения, naming conventions и delivery-политику.

Проектно-специфичные зависимости, auth-схемы, env-префиксы и release-политики должны жить в документации конкретного проекта, а не в этом пакете.
